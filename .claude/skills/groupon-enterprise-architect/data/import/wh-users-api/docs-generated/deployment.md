# Deployment

## Overview

wh-users-api is containerized using Docker and deployed to Kubernetes clusters across GCP and AWS regions. Deployments are managed by DeployBot (`deploy_kubernetes:v2.8.5-3.13.2-3.0.1-1.29.0`) using Helm chart `cmf-jtier-api` (version 3.88.1) rendered via `helm template` and applied with `krane`. The CI/CD pipeline is Jenkins-based (`jtierPipeline`). Staging deployments automatically promote to production.

## CI/CD Pipeline

- **Tool**: Jenkins (`jtierPipeline` shared library `java-pipeline-dsl@latest-2`)
- **Config**: `Jenkinsfile`
- **Trigger**: On push to `main` or `release` branch; deployments triggered via DeployBot
- **Releasable branches**: `main`, `release`
- **Deploy target (auto)**: `staging-us-central1`

### Pipeline Stages

1. **Build**: Compiles Java source and runs unit tests via `mvn clean verify`
2. **Code quality**: PMD static analysis (`pmd-rulesets.xml`, `exclude-pmd.properties`), FindBugs (`findbugs-exclude.xml`), JaCoCo coverage check
3. **Docker build**: Builds container image tagged with build version; pushes to `docker-conveyor.groupondev.com/mxp/wh-users-api`
4. **Deploy to staging**: DeployBot deploys to `staging-us-central1` (and `staging-europe-west1` independently)
5. **Promote to production**: After staging validation, DeployBot promotes `staging-us-central1` to `production-us-central1` and `staging-europe-west1` to `production-europe-west1`; `production-europe-west1` then promotes to `production-eu-west-1`

### Deployment Command

```bash
helm3 template cmf-jtier-api \
  --repo http://artifactory.groupondev.com/artifactory/helm-generic/ \
  --version '3.88.1' \
  -f .meta/deployment/cloud/components/api/common.yml \
  -f .meta/deployment/cloud/secrets/cloud/<env-region>.yml \
  -f .meta/deployment/cloud/components/api/<env-region>.yml \
  --set appVersion="${DEPLOYBOT_PARSED_VERSION}" \
  --set changeCause="${DEPLOYBOT_LOGBOOK_TICKET}" \
| krane deploy wh-users-api-<env> <kube-context> --global-timeout=300s --filenames=- --no-prune
```
