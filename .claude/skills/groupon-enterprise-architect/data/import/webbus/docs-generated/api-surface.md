# API Surface

## Overview

Webbus exposes a versioned REST API over HTTP. All production traffic enters via the `webbus.groupon.com` external VIP, which is IP-whitelisted to Salesforce. Every request must carry a valid `client_id` query parameter; invalid or missing client IDs return a `404` (obfuscated intentionally to prevent client enumeration). The single functional endpoint accepts bulk message payloads for publication to the Message Bus.

Endpoint documentation is auto-generated by Grape and served at the root path (`/`) of any running host.

## Request/Response Patterns

### POST /v2/messages/ — Request body

```json
{
  "messages": [
    {
      "topic": "jms.topic.salesforce.opportunity.create",
      "body": "<serialized-payload>"
    }
  ]
}
```

Both `topic` and `body` are required, non-empty strings for every element in the `messages` array.

### POST /v2/messages/ — Success response

HTTP `200` with an empty body when all messages publish successfully.

### POST /v2/messages/ — Partial failure response

HTTP `400` with a JSON body listing all messages that failed to publish. Salesforce redelivers any returned messages.

```json
[
  {
    "topic": "jms.topic.salesforce.opportunity.create",
    "body": "<serialized-payload>"
  }
]
```

### Common headers

- `Content-Type: application/json` — required for POST requests carrying a JSON body.
- `X-Request-Id` — propagated through all log entries via the `rack-request-id` middleware.

### Error format

Errors thrown via Grape's `throw :error` mechanism return the status code and a plain or JSON message body depending on the route context. Client ID validation failures deliberately return `404` rather than `401` or `403` to avoid disclosing whether a given ID exists. Message validation failures return `400` with `"<param>: must be a valid message"`.

### Pagination

> Not applicable — the endpoint accepts an unbounded array of messages per request but does not paginate responses.

## Rate Limits

No rate limiting is configured within Webbus itself. Rate control is enforced upstream by Salesforce, which throttles its own outbound message delivery. The SLA target is `< 2 s` per request payload at approximately 6 MB (~80 messages per request).

## Versioning

API versioning uses the URL path prefix strategy. The current version is `v2`, expressed as `/v2/messages/`. The version is declared in the Grape API definition using `:using => :path`.
