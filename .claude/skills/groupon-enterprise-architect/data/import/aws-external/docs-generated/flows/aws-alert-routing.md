# AWS Alert Routing to Cloud SRE

## Summary

When an AWS-related alert or incident signal is generated by Groupon's AWS control plane, the `aws-external` service metadata repository provides the ownership mapping that routes the alert to the Cloud SRE team. The routing is based on the `continuumAlertRoutingMetadata` component, which contains ownership definitions, informed by the `continuumRunbookReferenceIndex` that supplies operational context. The net result is that alerts originating from AWS activity land on the Cloud SRE team's queue rather than going unowned.

## Trigger

- **Type**: event
- **Source**: `continuumAwsControlPlane` — the AWS control plane component that monitors AWS activity and generates alerts and incident signals
- **Frequency**: On-demand, triggered whenever an AWS event produces an alert

## Participants

| Participant | Role in Flow | Architecture Ref |
|-------------|-------------|-----------------|
| AWS Control Plane | Generates the initial alert or incident signal | `continuumAwsControlPlane` |
| aws-external | Provides ownership identity and routing metadata | `continuumAwsExternal` |
| Alert Routing Metadata | Defines the Cloud SRE ownership mapping for AWS alerts | `continuumAlertRoutingMetadata` |
| Runbook Reference Index | Supplies operational context (onboarding, FAQ, troubleshooting links) that enriches the routing definition | `continuumRunbookReferenceIndex` |
| Cloud SRE Operations | Receives the routed alert with owner assignment and runbook context | `continuumCloudSreOperations` |

## Steps

1. **Generates incident signal**: AWS Control Plane detects an AWS event or anomaly and generates an alert or incident signal.
   - From: `continuumAwsControlPlane`
   - To: `continuumAwsExternal`
   - Protocol: Internal signal (alert routing)

2. **Resolves ownership mapping**: Alert Routing Metadata is consulted to identify the Cloud SRE team as the owner for this category of AWS alert.
   - From: `continuumAwsExternal`
   - To: `continuumAlertRoutingMetadata`
   - Protocol: Internal reference

3. **Enriches routing with runbook context**: Runbook Reference Index provides operational links (onboarding, FAQ, troubleshooting) to enrich the ownership mapping with actionable response guidance.
   - From: `continuumRunbookReferenceIndex`
   - To: `continuumAlertRoutingMetadata`
   - Protocol: Internal reference

4. **Routes alert with owner and context**: The enriched alert — containing owner assignment and runbook references — is routed to Cloud SRE Operations.
   - From: `continuumAwsExternal`
   - To: `continuumCloudSreOperations`
   - Protocol: Internal routing

## Error Handling

| Error Scenario | Handling Strategy | Outcome |
|---------------|-------------------|---------|
| Alert has no matching ownership mapping | Alert remains unrouted; Cloud SRE should be contacted directly | Manual triage required; update routing metadata to cover new alert type |
| AWS Control Plane unreachable | Alert signal is not generated; AWS degradation may go undetected | Monitor AWS Service Health Dashboard directly; rely on CloudWatch alarms |
