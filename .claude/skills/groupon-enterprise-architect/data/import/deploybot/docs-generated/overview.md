# deploybot Overview

## Purpose

deploybot is a SOX-compliant deployment orchestration service that wraps deployment operations (Capistrano, UberDeploy, Napistrano, Kubernetes) with audit trails, validation gates, and security controls. It enforces structured deployment workflows — including build validation, manual approval gates, and environment promotion — while automatically creating and closing Jira SOX logbook tickets for every deployment. The service serves as the authoritative deployment hub for Groupon's engineering teams, bridging GitHub push events to actual infrastructure changes.

## Scope

### In scope

- Receiving and validating GitHub push webhook events to trigger deployments
- Accepting direct programmatic deployment requests via REST API
- Executing deployments via Kubernetes API and Docker Engine
- Running the full deployment validation pipeline (build checks, manual approvals, ProdCAT, GPROD)
- Creating and closing SOX-compliant Jira logbook tickets for every deployment
- Archiving deployment logs to AWS S3
- Publishing deployment lifecycle events to PEST
- Sending Slack notifications at each deployment stage
- Supporting environment promotion (promoting a successful deploy to the next environment)
- Validating `.deploy_bot.yml` configuration files without performing a deployment
- Providing a web UI for deployment status and live log streaming
- Headless automated deployments without human approval
- OAuth2/OIDC authentication via Okta for protected actions (kill, authorize, promote)

### Out of scope

- Container image builds (handled by Conveyor CI / Jenkins)
- Infrastructure provisioning (handled by separate platform tooling)
- Service discovery and routing (handled by Kubernetes and platform networking)
- Artifact storage beyond log archival (handled by Artifactory)
