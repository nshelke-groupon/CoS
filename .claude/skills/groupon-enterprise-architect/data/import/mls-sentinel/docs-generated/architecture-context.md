# Architecture Context

## System Context

MLS Sentinel sits within the **Continuum** platform as a message-processing middleware layer. It bridges the legacy MessageBus event infrastructure and the Kafka-based MLS Command pipeline. Domain events generated by commerce systems (voucher service, CLO platform, Salesforce CRM) arrive via MessageBus; Sentinel validates each event against authoritative read-only upstream services, then emits typed Kafka Command messages. MLS Yang and other MLS read-model services subscribe to those Kafka topics to maintain their projections. Sentinel also owns a write API for the Merchant History domain.
