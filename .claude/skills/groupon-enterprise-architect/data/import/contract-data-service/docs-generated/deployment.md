# Deployment

## Overview

Contract Data Service is containerized using Docker and orchestrated on Google Cloud Platform (GCP) Kubernetes clusters via Groupon's Conveyor Cloud platform. The service runs in two environments: staging (`us-central1`, GCP stable VPC) and production (`us-central1`, GCP prod VPC). Both environments use SOX-scoped Kubernetes namespaces. Deployments are managed by Deploybot with a staging-to-production promotion workflow. CI/CD is handled by Jenkins using the JTier java-pipeline-dsl shared library.

## CI/CD Pipeline

- **Tool**: Jenkins
- **Config**: `Jenkinsfile`
- **Trigger**: On push to `main` (sox-inscope org); on push to `staging` branch (finance-engineering org)
- **Shared library**: `java-pipeline-dsl@latest-2`
- **Slack notifications**: `#contract-service-notifications`
- **Merge policy**: 2 ship-its required via `ghe-bot` (mergebot approval-by-comment, rebase merge)

### Pipeline Stages

1. **Build**: `mvn -U -B clean verify` — compiles, runs unit tests, integration tests, and code quality checks (PMD, FindBugs, JaCoCo)
2. **Docker build**: Builds application Docker image using `src/main/docker/Dockerfile`
3. **Push**: Pushes image to `docker-conveyor.groupondev.com/sox-inscope/contract-data-service`
4. **Deploy to staging**: Automatic deploy to `staging-us-central1` when build passes on `main` branch (sox-inscope org)
5. **Promote to production**: Manual promotion from staging via [Deploybot](https://deploybot.groupondev.com/sox-inscope/contract-data-service)

### Deploy Command

```bash
bash ./.meta/deployment/cloud/scripts/deploy.sh <target> <env> <namespace>
# Example staging:
bash ./.meta/deployment/cloud/scripts/deploy.sh staging-us-central1 staging contract-data-service-staging-sox
# Example production:
bash ./.meta/deployment/cloud/scripts/deploy.sh production-us-central1 production contract-data-service-production-sox
```
