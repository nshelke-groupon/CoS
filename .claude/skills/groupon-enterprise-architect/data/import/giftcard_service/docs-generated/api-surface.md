# API Surface

## Overview

The Giftcard Service exposes a REST JSON API under the `/api/v1/` namespace. All business endpoints accept and return JSON. Callers pass a country code in the `x-country-code` request header to determine the redemption currency. The service also exposes health and status endpoints for load-balancer and operational use. An OpenAPI/Swagger JSON schema is maintained at `swagger/giftcard_service.json` and is published to the service registry at `https://services.groupondev.com/services/giftcard_service/sha/HEAD/schema.json`.

## Endpoints

### Redemptions

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| `POST` | `/api/v1/redemptions` | Redeems a gift card and allocates Groupon Bucks to the user | Internal network |

**Request body parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `uuid` | string | yes | The user's UUID |
| `giftcard_number` | string | yes | The gift card number (16-digit for external, 22-char `vs-` prefix for internal VIS codes) |
| `giftcard_pin` | string | conditional | The 8-digit PIN (required for external First Data cards; omitted for VIS and Groupon credit codes) |
| `locale` | string | no | Locale string (e.g., `en_US`) used to derive country code for expiry calculations |

**Request headers:**

| Header | Type | Description |
|--------|------|-------------|
| `x-country-code` | string | Country code used to determine redemption currency (e.g., `US`, `DE`, `UK`). Defaults to USD if absent |
| `x-request-id` | string | Correlation ID; auto-generated by `RequestId` middleware if not provided |

**Response fields:**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"OK"` on success, `"Failed"` on error |
| `message` | string | Human-readable result or error detail |
| `amount` | number | Bucks amount credited to the user |
| `currency_code` | string | Currency code of the allocated bucks |
| `code` | string | Error classification code (4xx/5xx only): `UNAUTHORIZED_GIFTCARD`, `UNAUTHORIZED_EXTERNAL_GIFTCARD`, `UNAUTHORIZED_INTERNAL_GIFTCARD`, `INTERNAL_GIFTCARD_LIMITS`, `INVALID_REQUEST`, `SERVER_ERROR` |
| `errors` | object | Error detail map (4xx/5xx only) |

### Legacy Credit Codes

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| `POST` | `/api/v1/legacy_credit_codes` | Creates a batch of legacy currency credit codes for a campaign | Internal network |

**Request body parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `campaign_name` | string | yes | Name of the campaign the codes belong to |
| `value_in_minor_unit` | integer | yes | Value of each credit code in the minor currency unit (e.g., cents) |
| `expiry_date` | string | yes | Expiry date for the codes (datetime string) |
| `quantity` | integer | yes | Number of credit codes to create |

**Request headers:**

| Header | Type | Description |
|--------|------|-------------|
| `x-country-code` | string | Required. Country code used to determine credit code currency |

**Response fields:**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"OK"` on success, `"Failed"` on error |
| `legacy_credit_codes` | array | Array of created credit code strings (10-character alphanumeric) |
| `code` | string | Error code on failure: `INVALID_REQUEST` or `SERVER_ERROR` |
| `errors` | object | Error detail map (4xx only) |

## Request/Response Patterns

### Common headers
- `x-country-code` — determines currency for all business endpoints
- `x-request-id` — correlation ID propagated through logs; generated automatically if absent

### Error format
All 4xx and 5xx responses follow the shape:
```json
{
  "status": "Failed",
  "message": "<human-readable description>",
  "code": "<ERROR_CODE>",
  "errors": { "<field>": "<error detail>" }
}
```
