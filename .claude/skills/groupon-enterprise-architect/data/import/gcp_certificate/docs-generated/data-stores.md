# Data Stores

## Overview

The `gcp_certificate` service does not own an application-level database or cache. Its only persistent storage is GCS buckets used to store Terraform remote state. The actual PKI data — CA certificates, certificate templates, and issuance records — is stored and managed entirely within GCP Private CA (the `google_privateca_ca_pool` and `google_privateca_certificate_authority` resources). Certificate private keys generated by requesters are never stored by this service.

#### Key Entities

| Entity / Table | Purpose | Key Fields |

#### Access Patterns

- **Read**: Terragrunt reads state on every `plan` and `apply` to detect drift
- **Write**: Terragrunt writes updated state after successful `apply` or `destroy`
- **Indexes**: Not applicable (object storage)

## Caches

> No evidence found in codebase. No application-level caching is used.

## Data Flows

GCS bucket names follow the pattern `grpn-gcp-<PROJECTNAME>-state-<GCP_PROJECT_NUMBER>`. The bucket location is `US` (multi-region). The prefix within the bucket corresponds to the relative Terragrunt module path (e.g., `us-central1/private-ca`, `us-central1/certificates`). State is initialized per-module and not shared across modules.
