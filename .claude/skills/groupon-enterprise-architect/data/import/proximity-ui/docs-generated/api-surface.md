# API Surface

## Overview

Proximity UI exposes a REST API under the `/api/proximity/` path namespace. These endpoints are consumed exclusively by the Vue.js SPA running in the same browser session. Each endpoint is a thin proxy: the Express server receives the browser request, forwards it to `continuumProximityHotzoneApi` (appending `client_id=ec-team`), and relays the response verbatim. No data transformation occurs on the server side. All other GET routes return `index.html` to support SPA client-side routing.

## Endpoints

### Users

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/api/proximity/users` | Returns the list of allowed users from the Hotzone API | `X-Remote-User` header (set by Nginx) |
| GET | `/api/proximity/users/current` | Returns the currently authenticated user's username from the `X-Remote-User` request header | `X-Remote-User` header |

### Campaigns

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/api/proximity/campaigns` | Lists all deal-based hotzone campaigns | `X-Remote-User` header |
| GET | `/api/proximity/campaigns/:campaignId` | Retrieves a single campaign by ID | `X-Remote-User` header |
| POST | `/api/proximity/campaigns` | Creates a new campaign | `X-Remote-User` header |
| POST | `/api/proximity/campaigns/:campaignId` | Updates an existing campaign | `X-Remote-User` header |
| DELETE | `/api/proximity/campaigns/:campaignId` | Deletes a campaign by ID | `X-Remote-User` header |

### Categories

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/api/proximity/categories` | Retrieves the list of deal categories from the Hotzone API | `X-Remote-User` header |

### Hotzone Deals

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| POST | `/api/proximity/hotzoneDeals` | Creates one or more hotzone deals (supports bulk via `hotZoneDeals` array payload) | `X-Remote-User` header |
| POST | `/api/proximity/hotzoneDeals/browse` | Searches/browses existing hotzones (server-side DataTables pagination) | `X-Remote-User` header |
| GET | `/api/proximity/hotzoneDeals/:hotzoneId` | Retrieves a single hotzone deal by ID including deal info, hours, and geo-point | `X-Remote-User` header |
| DELETE | `/api/proximity/hotzoneDeals/:hotzoneId` | Deletes a hotzone deal by ID | `X-Remote-User` header |

## Request/Response Patterns

### Common headers

- `X-Remote-User`: Set by the upstream Nginx reverse proxy. Used to identify the current user on the `/api/proximity/users/current` endpoint.
- `x-forwarded-for`: Forwarded to the upstream Hotzone API on GET proxy calls for IP logging.
- `Content-Type: application/json`: All proxy requests to the Hotzone API use JSON mode via the `request` library's `json: true` option.

### Error format

Upstream error responses from `continuumProximityHotzoneApi` are relayed directly to the browser with the same HTTP status code and body. The server adds a `500` response with the string `'Error'` for uncaught Express errors.

### Pagination

The `/api/proximity/hotzoneDeals/browse` endpoint supports server-side DataTables pagination. The browser POST body includes DataTables draw parameters; the response is generated by the upstream Hotzone API's `/browse` endpoint.

## Rate Limits

No rate limiting configured.

## Versioning

No API versioning is applied to the `/api/proximity/*` namespace. The upstream Hotzone API uses `/v1/proximity/location/hotzone` paths, but this is an internal implementation detail of the proxy configuration.
