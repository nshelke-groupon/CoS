# Architecture Context

## System Context

OpenVPN Config Automation sits within the Groupon Continuum platform as the operational management layer for the OpenVPN Cloud Connexa ZTNA service. Operators (InfoSec and NetOps engineers) invoke Python CLI scripts directly; those scripts call the OpenVPN Cloud Connexa SaaS API over HTTPS to read and write network access configuration. The automation is supported by three external identity and infrastructure dependencies: Okta (user identity provider), corpad (corporate directory), and AWS DNS (network connector DNS). Local JSON backup files stored in the `backup/` directory act as the persistence layer between export and restore operations.

## Components by Container

### OpenVPN Config Automation (`openvpnConfigAutomation`)

| Component | Responsibility | Technology |
|-----------|---------------|-----------|
| OpenVPN API Client | Shared OAuth 2.0 token acquisition, rate-limit-aware request wrapper, and paginated entity listing | Python module (`openvpn_api.py`) |
| Delete User Job | CLI entry point that resolves a user by email address and issues a DELETE request to the Cloud Connexa API | Python script (`delete_user.py`) |
| Export Backup Job | CLI entry point that iterates all entity types via paginated API reads and writes them as JSON to `backup/` | Python script (`export_backup.py`) |
| Restore Backup Job | CLI entry point that reads JSON backup files and recreates networks, routes, IP services, apps, and access groups in Cloud Connexa | Python script (`restore_backup.py`) |
| Backup Storage Adapter | File read/write routines for the local `backup/` directory, used by both export and restore jobs | Filesystem adapter (inline in export/restore scripts) |
