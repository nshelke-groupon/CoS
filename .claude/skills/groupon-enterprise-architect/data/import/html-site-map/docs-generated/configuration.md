# Configuration

## Overview

html-site-map uses a layered CSON configuration system managed by `keldor-config`. Configuration is assembled at startup by layering files in order of increasing specificity: base defaults, then stage-specific overrides (driven by `KELDOR_CONFIG_SOURCE`), then optional region-specific overrides, and finally node-environment overrides (`NODE_ENV`). An optional `config/local.cson` file (not checked in) can be used for local developer overrides. Deployment-specific runtime environment variables are injected via Kubernetes pod spec, generated by `napistrano` into `.deploy-configs/*.yml`.

## Environment Variables

| Variable | Purpose | Required | Default | Source |
|----------|---------|----------|---------|--------|
| `NODE_ENV` | Node.js environment mode (`production`, `staging`, `development`, `test`) | yes | — | Kubernetes / shell |
| `KELDOR_CONFIG_SOURCE` | Selects the stage config overlay file (`{production}`, `{staging}`) | yes | — | `.deploy-configs/*.yml` (helm/env) |
| `PORT` | HTTP port the itier-server listens on | yes | `8000` | `.deploy-configs/*.yml` |
| `NODE_OPTIONS` | Node.js runtime options; controls max heap size | yes | `--max-old-space-size=512` (GCP) / `--max-old-space-size=1024` (AWS us-west-1) | `.deploy-configs/*.yml` |
| `UV_THREADPOOL_SIZE` | libuv thread pool size for async I/O | yes | `75` | `.deploy-configs/*.yml` |

## Per-Environment Overrides

Configuration layering order (each step overrides the previous):

1. **`config/base.cson`** — universal defaults applied in all environments (API proxy client ID, steno transport)
2. **`config/stage/{production|staging}.cson`** (selected by `KELDOR_CONFIG_SOURCE`) — CDN host and service client base URL differ between production and staging
3. **Region-specific overrides** (e.g. `config/stage/production-us-central1.cson`) — avoided where possible per README guidance; no region-specific CSON files are currently present
4. **`config/node-env/{development|test|production}.cson`** (selected by `NODE_ENV`) — production enables file-based steno logging
5. **`config/local.cson`** (optional, not checked in) — local developer one-off overrides (e.g. pointing lpapi to staging endpoint, enabling console logging)

Key production-vs-staging differences:

| Setting | Staging | Production |
|---------|---------|------------|
| CDN hosts | `staging<1,2>.grouponcdn.com` | `www<1,2>.grouponcdn.com` |
| Service client base URL | `{staging}` | `{production}` |
| `KELDOR_CONFIG_SOURCE` | `{staging}` | `{production}` |
| `NODE_OPTIONS` heap limit | `512 MB` | `512 MB` (GCP) / `1024 MB` (AWS us-west-1) |
| Replicas | min 1, max 3 | min 1–2, max 3–5 (region-dependent) |
