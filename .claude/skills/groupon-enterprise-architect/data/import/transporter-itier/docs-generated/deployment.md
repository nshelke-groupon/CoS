# Deployment

## Overview

Transporter I-Tier is containerized using Docker and deployed to Groupon's Conveyor Cloud Kubernetes clusters across multiple AWS and GCP regions. Deployment artifacts are built by Conveyor CI (Jenkins) and deployed via Napistrano and Deploybot. The service runs behind Groupon's Hybrid Boundary ingress layer, which handles mTLS termination and routes internal service DNS names to the appropriate region VIPs.

## Infrastructure

| Component | Technology | Details |
|-----------|-----------|---------|
| Container | Docker | `Dockerfile` — base image: `docker-conveyor.groupondev.com/conveyor/alpine-node16.14.2:2022.04.20-16.19.18-4028cb7` |
| Orchestration | Kubernetes (Conveyor Cloud) | Manifests generated by Napistrano from `.deploy-configs/*.yml` via Helm chart `cmf-itier` version `3.88.1` |
| Load balancer | Hybrid Boundary | mTLS interceptor active; ingress and egress enabled per region; routes `transporter-itier.production.service` and `transporter-itier.staging.service` |
| CDN | Groupon CDN | `www<1,2>.grouponcdn.com` (production), `staging<1,2>.grouponcdn.com` (staging) for static assets |
| Log shipping | Filebeat + Kafka + ELK | Filebeat sidecar ships steno-format logs to Kafka; consumed by Logstash and indexed in Kibana |
| Metrics | Telegraf / Wavefront | Application metrics follow Standard Measurement Architecture and are published to Wavefront |

## CI/CD Pipeline

- **Tool**: Conveyor CI (Jenkins) — `Jenkinsfile` uses `@Library('itier-pipeline-dsl@latest-1')`
- **Config**: `Jenkinsfile` (CI pipeline), `.deploy-configs/*.yml` (CD via Napistrano), `.deploy_bot.yml` (Deploybot targets)
- **Trigger**: CI runs on every push; CD to staging triggered automatically on `master` and `staging` branches; production deploy is manual (Deploybot)
- **Slack notifications**: `#salesforce-deploy`

### Pipeline Stages

1. **Lint**: Runs ESLint over all JS source files via `npm run lint:js`
2. **Test (unit)**: Runs `c8 mocha 'modules/**/*.test.*'` with text and JSON-summary coverage reporters
3. **Test (integration)**: Runs `mocha test/integration` using Testium browser driver
4. **Build assets**: Compiles and bundles client-side JS/CSS modules via Webpack in production mode (`npm run dist-assets`)
5. **Docker build**: Builds container image and pushes to `docker-conveyor.groupondev.com/salesforce/transporter-itier`
6. **Deploy to staging**: Napistrano/Deploybot deploys to `staging-us-west-1` and `staging-us-central1`
7. **Promote to production**: Manual approval triggers Deploybot promotion from staging artifact to production regions

Rollback command:
```
npx nap --cloud rollback staging us-central1
```
