# Integrations

## Overview

Conveyor K8s integrates with six external systems. AWS and GCP are the primary cloud targets for cluster provisioning. AWS S3 stores cluster discovery state. Kubernetes clusters are the direct target of configuration management and readiness checks. Wavefront receives deployment event markers. Jenkins and GitHub Actions orchestrate pipeline execution. There are no internal Groupon service dependencies — this is a pure platform-layer component.

## External Dependencies

| System | Protocol | Purpose | Critical | Architecture Ref |
|--------|----------|---------|----------|-----------------|
| AWS EKS | SDK / Terraform provider | Provision and manage EKS Kubernetes clusters | yes | `conveyorK8sTerraformEks` |
| AWS EC2 | SDK (aws-sdk-go) | AMI lookup and cross-account AMI copy for worker node images | yes | `conveyorK8sAmiBaking`, `conveyorK8sPipelineUtils` |
| AWS S3 | SDK (aws-sdk-go) | Cluster state bucket discovery; Terraform EKS remote state backend | yes | `conveyorK8sPipelineUtils`, `conveyorK8sTerraformEks` |
| GCP GKE | Terraform provider (google, google-beta ~6.21.0) | Provision and manage GKE Kubernetes clusters | yes | `conveyorK8sTerraformGke` |
| GCP GCS | Terraform provider | Terraform GKE remote state backend | yes | `conveyorK8sTerraformGke` |
| Kubernetes API | k8s client-go (v0.28.0-alpha.3) | Readiness checks on LoadBalancer services; read/write promotion ConfigMaps | yes | `conveyorK8sPipelineUtils` |
| Wavefront | REST HTTP (POST `/api/v2/event`) | Create and close deployment events during cluster promotion windows | no | `conveyorK8sPipelineUtils` |

### AWS EKS / EC2 / S3 Detail

- **Protocol**: AWS SDK v1 (`github.com/aws/aws-sdk-go v1.44.251`) and Terraform AWS provider `~> 4.2.0`
- **Base URL / SDK**: `github.com/aws/aws-sdk-go`, `hashicorp/aws ~> 4.2.0`
- **Auth**: IAM roles (`grpn-all-conveyor-admin`, `grpn-sandbox-conveyor-dev`, `grpn-all-csre`) and IAM users (`conveyor-provisioner`, `conveyor_playbook_provisioner`) mapped in `terra-eks/modules/eks-cluster/variables.tf`
- **Purpose**: EKS cluster lifecycle (create, update, delete); EC2 AMI baking and cross-account publishing; S3 cluster state bucket discovery
- **Failure mode**: Pipeline fails fast; error is surfaced in Jenkins/GitHub Actions job log
- **Circuit breaker**: No — pipelines use retry logic in `services_readiness` binary (exponential backoff, configurable attempts)

### GCP GKE / GCS Detail

- **Protocol**: Terraform google/google-beta providers `~> 6.21.0`; GCP Service Account JSON auth
- **Base URL / SDK**: `hashicorp/google ~> 6.21.0`, `hashicorp/google-beta ~> 6.21.0`
- **Auth**: GCP Service Account JSON (`GCP_SERVICE_ACCOUNT_JSON` secret), applied via `conveyor-cloud/conveyor-gh-shared-actions/.github/actions/gcp-auth@main`
- **Purpose**: GKE cluster provisioning (create, update, node pool upgrade); GCS as Terraform state backend
- **Failure mode**: GitHub Actions job fails; Ansible deploy step returns non-zero exit code
- **Circuit breaker**: No

### Kubernetes API Detail

- **Protocol**: k8s client-go REST over HTTPS
- **Base URL / SDK**: `k8s.io/client-go v0.28.0-alpha.3`; `github.groupondev.com/conveyor-cloud/go-cluster-libraries v0.0.10`
- **Auth**: Kubeconfig generated by EKS/GKE auth; `svc_conveyor` Ansible user
- **Purpose**: Service readiness verification (LoadBalancer services comparison between source and destination clusters); ConfigMap read/write for promotion metadata
- **Failure mode**: `services_readiness` binary returns non-zero; pipeline stops promotion
- **Circuit breaker**: Exponential backoff via `github.com/jpillora/backoff v1.0.0`, up to configurable max attempts

### Wavefront Detail

- **Protocol**: REST HTTP `POST /api/v2/event`, `POST /api/v2/event/{id}/close`
- **Base URL / SDK**: Configured at runtime via `baseUrl` and `apiToken` CLI flags; `wavefront` binary in `conveyor-pipeline-utils`
- **Auth**: Bearer token (`Authorization: Bearer <apiToken>`)
- **Purpose**: Mark promotion start/end in Wavefront for metrics correlation
- **Failure mode**: Logged but non-fatal — promotion continues
- **Circuit breaker**: No (5-second HTTP timeout only)

## Internal Dependencies

> No evidence found in codebase. Conveyor K8s does not call any other internal Groupon microservices directly.

## Consumed By

> Upstream consumers are tracked in the central architecture model. This service is invoked by:
>
> - Jenkins pipeline jobs in the `conveyor-cloud` Jenkins folder (e.g., `conveyor-cloud/gke/apply-playbook`, `conveyor-cloud/gke/manage-gke-cluster`)
> - GitHub Actions workflows triggered on pull request, version tag push, or `workflow_dispatch`
> - All Groupon service teams indirectly (they run on clusters managed by this repo)

## Dependency Health

- **AWS**: No health check mechanism configured; pipeline failures surface as job errors
- **GCP**: GitHub Actions workflow fails if GCP authentication fails; no explicit health-check probe
- **Kubernetes API**: `services_readiness` binary polls with configurable backoff (default: 5 attempts, 1–5 minute intervals, factor 1.5, jitter enabled)
- **Wavefront**: Fire-and-forget; non-zero HTTP status is logged but does not halt the pipeline
