# Deployment

## Overview

Alligator is containerized (Docker) and deployed to Kubernetes clusters via Raptor/Conveyor on GCP. It is also maintained on legacy on-premises infrastructure (JTier on bare-metal, colos sac1/snc1/dub1) via Capistrano. The CI pipeline runs on Cloud Jenkins. Releases are promoted through DeployBot. The service operates across 10 distinct deployment targets spanning US and EMEA regions at staging, beta (canary), and production tiers.

## CI/CD Pipeline

- **Tool**: Jenkins (Cloud Jenkins)
- **Config**: `Jenkinsfile` (uses `java-pipeline-dsl@latest-2` shared library) and `.ci.yml`
- **Trigger**: Push to `main` branch triggers build and staging deployment; pull request branches trigger build only
- **DeployBot**: https://deploybot.groupondev.com/relevance/alligator
- **Jenkins URL**: https://cloud-jenkins.groupondev.com/job/relevance/job/alligator/

### Pipeline Stages

1. **Build**: `mvn clean package` compiles the application, runs unit tests, and produces the JAR artifact
2. **OpenAPI generation**: `mvn -DskipDocker verify` generates the OpenAPI 3.0.3 spec to `doc/swagger/`
3. **Docker image build**: Packages the JAR into the `docker-conveyor.groupondev.com/relevance/alligator-service` image using `src/main/docker/Dockerfile`
4. **Publish**: Pushes the artifact to Artifactory and the Docker image to the Conveyor registry
5. **Deploy to staging**: On successful `main` build, DeployBot creates a pending release for promotion to `staging-us-central1` and `staging-europe-west1`
6. **Beta (canary) promotion**: Operator-initiated via DeployBot; promotes to `beta-us-central1`, `beta-europe-west1`, `beta-eu-west-1`
7. **Production promotion**: Operator-initiated via DeployBot; promotes from beta to `production-us-central1`, `production-europe-west1`, `production-eu-west-1`

Alpha and beta releases can also be triggered by pushing tags matching `alpha-<major>.<minor>-*` or `beta-<major>.<minor>-*` (see `Jenkinsfile` `ALPHA_RELEASE_CONFIG` / `BETA_RELEASE_CONFIG`).
