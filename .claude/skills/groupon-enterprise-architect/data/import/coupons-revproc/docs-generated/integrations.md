# Integrations

## Overview

coupons-revproc has four external dependencies and two internal platform dependencies. The most critical external dependency is AffJet (the source of all affiliate transaction data). VoucherCloud API provides click enrichment and redirect data. Salesforce receives bonus payment submissions. Dotidot SFTP receives coupon feed files. Internally the service depends on Groupon's Mbus (JMS) for downstream event publishing, and MySQL/Redis as owned data stores described in [Data Stores](data-stores.md).

## External Dependencies

| System | Protocol | Purpose | Critical | Architecture Ref |
|--------|----------|---------|----------|-----------------|
| AffJet API | HTTPS (REST / Retrofit2) | Source of affiliate transaction data; paginated per country and date range | yes | `voucherCloudApi` (stub in local DSL; separate AffJet endpoint) |
| VoucherCloud API | HTTPS (REST / Retrofit2) | Click enrichment, merchant lookup, offer details, redirect URL fetching | yes | `voucherCloudApi` |
| Salesforce | HTTPS (REST via SalesforceHttpClient 1.16) | Bonus payment submission and reconciliation updates | no | `salesForce` |
| Dotidot SFTP | SFTP (JSch 0.2.17) | Upload of coupon feed files; egress to IP 178.77.214.157 port 22 | no | external SFTP server |

### AffJet API Detail

- **Protocol**: HTTPS, Retrofit2 with OkHttp
- **Base URL / SDK**: Configured in `affjet.httpClients` block of JTIER_RUN_CONFIG YAML; per-country HTTP clients defined in `AffJetHttpClients`
- **Auth**: API key / credentials configured in secrets; enabled flag in `affjet.enabled` config key
- **Purpose**: The primary transaction data source. The service supports 19 country/network variants: US, GB, AU, DE, ES, FR, IT, NL, PL, IE, VC_GB, VC_AU, VC_BR, VC_DE, VC_FR, VC_IE, VC_NL, WL_GB, WL_IE (see `SupportedAffJetCodes`). Ingestion queries a 30-day rolling window, paged by date and page index.
- **Failure mode**: Ingestion job logs error and returns failure count; transactions are not written for the failed pages; Quartz will retry on next scheduled run
- **Circuit breaker**: No evidence found in codebase

### VoucherCloud API Detail

- **Protocol**: HTTPS, Retrofit2 with OkHttp
- **Base URL / SDK**: Configured in `voucherCloudApi` Retrofit config block in JTIER_RUN_CONFIG YAML; `VoucherCloudRetrofitApi` defines endpoints
- **Auth**: Credentials configured in secrets; passed as HTTP headers or query params by the Retrofit client
- **Purpose**: Fetches click details (click ID, attribution ID, bcookie, deal UUID, source, event ID), merchant lists, merchant offers, and redirect URL mappings for cache prefill and sanitization
- **Failure mode**: `VoucherCloudException` is thrown; `TransactionProcessor` handles the absence of a click gracefully (builds transaction without click enrichment)
- **Circuit breaker**: No evidence found in codebase

### Salesforce Detail

- **Protocol**: HTTPS REST, via internal `SalesforceHttpClient` library (version 1.16)
- **Base URL / SDK**: `salesforce.url` / `salesforce.urlHost` / `salesforce.urlScheme` in JTIER_RUN_CONFIG YAML; OAuth2 credentials in `salesforce.clientId`, `salesforce.clientSecret`, `salesforce.redirectURI`, `salesforce.username`, `salesforce.password`, `salesforce.securityToken`
- **Auth**: OAuth2 (username + password + security token flow via SalesforceHttpClient)
- **Purpose**: Sends bonus payment records and reconciliation updates for processed coupon transactions
- **Failure mode**: `SalesforceException` is thrown; errors are logged
- **Circuit breaker**: No evidence found in codebase

### Dotidot SFTP Detail

- **Protocol**: SFTP via JSch 0.2.17
- **Base URL / SDK**: `dotidot.sftpCredentials.host` / `dotidot.sftpCredentials.port` / `dotidot.sftpCredentials.user` / `dotidot.sftpCredentials.password` in JTIER_RUN_CONFIG YAML; target IP 178.77.214.157 port 22 (network policy egress rule in `coupons-feed-generator` deployment)
- **Auth**: SFTP username/password (credentials stored in secrets)
- **Purpose**: Upload of coupon feed files generated by `CouponsFeedExporter` on a daily cron schedule (04:00 UTC)
- **Failure mode**: SFTP upload errors are logged; feed regeneration retried on next scheduled run
- **Circuit breaker**: No evidence found in codebase

## Consumed By

> Upstream consumers are tracked in the central architecture model. The `GET /transactions` endpoint is called by internal Groupon services querying processed transaction data. Manual ingestion triggers are called by operators or internal tooling.

## Dependency Health

- AffJet and VoucherCloud failures are logged as structured Steno log events. No automated circuit breakers are configured; recovery relies on the next Quartz scheduled execution.
- Salesforce failures raise `SalesforceException` logged at error level.
- Mbus send failures are caught as `MessageException` and logged at error level; the persisted transaction is not rolled back.
- SFTP upload failures are logged; the feed cron job will retry at its next scheduled window.
