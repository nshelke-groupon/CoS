# Data Stores

## Overview

The plugin owns a small dedicated MySQL schema within the Kill Bill database. It stores short-lived authorization tokens used during GAPI order creation, and retry notification queue records used to reconcile payment updates. The Kill Bill core tables (accounts, subscriptions, invoices, custom fields) are owned and managed by the Kill Bill platform; the plugin reads and writes to those via the Kill Bill OSGi API rather than direct SQL.

#### Key Entities

| Entity / Table | Purpose | Key Fields |
|----------------|---------|-----------|
| `sp_token` | Stores short-lived authorization tokens used to authenticate GAPI order creation calls | `user_id`, `subscription_id`, `token`, `expiration_time` |
| `sp_notifications` | Kill Bill notification queue table for scheduling MBus retry jobs | `search_key1` (accountId), `effective_date`, `queue_name`, `event_json` |
| `sp_notifications_history` | History/audit table for processed notification queue entries | Same structure as `sp_notifications` |

#### Access Patterns

- **Read**: Token lookup by `user_id` + `subscription_id` during order creation; notification queue polling for retry processing
- **Write**: Token insert/update on each order creation attempt (token expiration time: `com.groupon.volume2.sp.plugin.token.expirationTimeMillisSec`, default 120000 ms); notification queue inserts on MBus event processing failures
- **Indexes**: Primary key on `sp_token`; Kill Bill notification queue indexes on `search_key1`, `effective_date`, and `processing_state` (generated by Kill Bill notification queue framework)

## Data Flows

Kill Bill custom fields are used as a secondary metadata store (not owned by the plugin schema). When a subscription order is created or fails, the plugin writes the following custom fields via the Kill Bill API:

| Object Type | Custom Field Name | Value |
|-------------|------------------|-------|
| `SUBSCRIPTION` | `DEAL_ID` | Groupon deal UUID |
| `SUBSCRIPTION` | `OPTION_ID` | Groupon option UUID |
| `SUBSCRIPTION` | `BILLING_RECORD_ID` | Billing record UUID |
| `INVOICE` | `ORDER_ID` | Groupon order UUID (returned by `createOrder`) |
| `INVOICE` | `ORDER_FAILED` | Order error details (when `createOrder` fails) |

These custom fields are stored in Kill Bill's core tables and are the primary mechanism for correlating Kill Bill invoices with Groupon commerce orders.
