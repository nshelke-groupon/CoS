# Integrations

## Overview

The Merchant Booking Tool has two internal Continuum Platform dependencies and two external dependencies. Internally it relies on the Universal Merchant API for all booking data operations and the Layout Service for merchant shell rendering. Externally it integrates with Google OAuth for calendar sync authentication and Inbenta for embedded support knowledge flows. All integrations use synchronous HTTPS/JSON request/response patterns.

## External Dependencies

| System | Protocol | Purpose | Critical | Architecture Ref |
|--------|----------|---------|----------|-----------------|
| Google OAuth | OAuth 2.0 / HTTPS | Authenticates Google Calendar sync flows for merchants | no | `googleOAuth` |
| Inbenta | HTTPS | Provides embedded support knowledge base authentication; support tokens generated by `mbtInbentaClient` | no | Not in federated model (stub only) |

### Google OAuth Detail

- **Protocol**: OAuth 2.0 / HTTPS
- **Base URL / SDK**: Standard Google OAuth 2.0 authorization endpoints
- **Auth**: OAuth 2.0 authorization code flow
- **Purpose**: Enables merchants to connect their Google Calendar for booking synchronization
- **Failure mode**: Google Calendar sync unavailable; core booking management continues to function
- **Circuit breaker**: No evidence found in codebase

### Inbenta Detail

- **Protocol**: HTTPS
- **Base URL / SDK**: Inbenta API (exact base URL not in federated model)
- **Auth**: Token-based; `mbtInbentaClient` builds auth payloads and requests support tokens
- **Purpose**: Provides embedded merchant support and knowledge base functionality
- **Failure mode**: Embedded support flows unavailable; core booking management continues to function
- **Circuit breaker**: No evidence found in codebase

## Internal Dependencies

| Service | Protocol | Purpose | Architecture Ref |
|---------|----------|---------|-----------------|
| Universal Merchant API | HTTPS/JSON | Primary source of all booking-service data — reservations, calendars, accounts, campaigns, workshops, and staff profiles; both read and write operations | `continuumUniversalMerchantApi` |
| Layout Service | HTTPS | Renders merchant shell/layout and app context for I-tier page composition | `continuumLayoutService` |

### Universal Merchant API Detail

- **Protocol**: HTTPS/JSON
- **Purpose**: The `mbtMerchantApiClient` component calls booking-service endpoints for reservations, calendars, accounts, campaigns, workshops, and staff profiles. The `mbtProxyController` also forwards write/update operations directly to the booking service base URL.
- **Failure mode**: Booking management pages fail to render; merchant cannot manage reservations or calendar
- **Circuit breaker**: No evidence found in codebase

### Layout Service Detail

- **Protocol**: HTTPS
- **Purpose**: Provides the merchant shell and app context rendered around MBT page content
- **Failure mode**: Pages may render without merchant navigation shell
- **Circuit breaker**: No evidence found in codebase
