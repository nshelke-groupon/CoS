# pre_task_tracker Overview

## Purpose

`pre_task_tracker` is an Apache Airflow DAG package deployed to Google Cloud Composer that provides comprehensive monitoring, alerting, and operational automation for Groupon's data engineering platforms. It detects and alerts on Megatron data pipeline delays, failed or long-running Airflow tasks, idle or long-running Dataproc clusters, Teradata table staleness, and MBUS message queue backlogs. The service also automates SLA tracking record maintenance and runbook document creation from resolved incident tickets.

## Scope

### In scope

- Monitoring Airflow task states (failed, long-running, queued, skip-sequence) across `dnd-gcp-migration-data-eng`, `dnd-bia-data-engg`, and `consumer-data-engineering` organizations
- Detecting Megatron end-of-day (EOD) SLA breaches and intra-day lag for Teradata and Hive pipeline tables configured in `config.yml`
- Tracking idle and long-running Dataproc clusters across PIPELINES, CONSUMER, and INGESTION GCP projects
- Monitoring Teradata table limits for Magneto-dependent tables (`User_1`, `Account_1`, `Opportunity_1`)
- Monitoring MBUS queue backlogs and triggering catch-up pipeline DAGs when thresholds are exceeded
- Creating and resolving Jira Service Management (JSM) alerts for all of the above incident types
- Updating EDW/RM SLA dashboard entries in the CKOD MySQL database
- Automatically generating and updating runbook documents in Google Drive from resolved Jira tickets
- Cleaning up errored and paused DAG files from GCS buckets and tracking them in MySQL
- Sending a heartbeat to the Atlassian JSM integration to confirm service liveness

### Out of scope

- Executing or scheduling the data pipeline DAGs being monitored (those are owned by their respective teams)
- Managing Megatron pipeline source data or ETL logic
- Configuring SLA definitions in the CKOD database (handled by pipeline owners)
- User-facing HTTP API (no web server; all interactions via Airflow UI and JSM)

## Domain Context

- **Business domain**: Platform Reliability Engineering — operational monitoring and alerting for Groupon data pipelines
- **Platform**: Google Cloud Platform (Cloud Composer, Dataproc, GCS, Secret Manager)
- **Upstream consumers**: Airflow scheduler (triggers DAGs on schedule), Jira Service Management (receives alerts and heartbeats), Google Drive (receives runbook documents)
- **Downstream dependencies**: Airflow PostgreSQL metadata database, PRE Monitoring MySQL database, Megatron MySQL database (`megatron` conn), DWH Manage MySQL database (`dwh_manage` conn), Teradata (`dwh_load_sf` schema), Jira (`GPROD` project), Google Cloud Secret Manager, Google Chat webhook, Grafana/Prometheus API
