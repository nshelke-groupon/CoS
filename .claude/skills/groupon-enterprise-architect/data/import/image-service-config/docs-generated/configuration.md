# Configuration

## Overview

All configuration for the Image Service is file-based. There are no environment variables or external config stores (Consul, Vault, Kubernetes ConfigMaps) referenced in this repository. Configuration is distributed to target nodes by Capistrano tasks that SCP files to app and cache nodes. Two primary config files exist: `config.yml` (client API keys and allowed image sizes) and `nginx.conf` / ERB templates (nginx proxy behavior). Environment-specific values are injected at deploy time via ERB templating in `nginx.conf.erb` and `upstream-default.conf.erb`.

## Config Files

| File | Format | Purpose |
|------|--------|---------|
| `config.yml` | YAML | Master client registry: S3 credentials, CDN hostname, client API keys, and per-client allowed image-size whitelists. Config bundle version (`1.6.0`). |
| `consolidated_config.yml` | YAML | Auto-generated by `consolidate_all_allowed_sizes.py`; consolidates all client allowed-size lists into a single shared set. Used to understand the union of all permissible image dimensions. |
| `nginx.conf` | Nginx | Production nginx configuration (static, committed). Defines worker count (8), worker connections (16384), proxy buffer sizes, proxy cache settings, and virtual host server blocks. |
| `nginx.conf.erb` | ERB / Nginx | ERB template for nginx.conf; interpolates `server_names_cdn`, `server_names_proxy`, `proxy_set_header`, and DNS resolver addresses. Rendered by Capistrano at deploy time. |
| `upstream-default.conf` | Nginx | Default upstream definition pointing to `global-image-service-app-vip:80`. Active in production steady-state. Symlinked to `/etc/nginx/upstream.conf`. |
| `upstream-default.conf.erb` | ERB / Nginx | ERB template for upstream.conf; interpolates `upstream_vip` with environment-specific VIP address. |
| `upstream-app1-disabled.conf` | Nginx | Upstream definition routing to only app2, app3, app4 nodes (app1 excluded). Used during rolling restarts. |
| `upstream-app2-disabled.conf` | Nginx | Upstream definition routing to only app1, app3, app4 nodes (app2 excluded). |
| `upstream-app3-disabled.conf` | Nginx | Upstream definition routing to only app1, app2, app4 nodes (app3 excluded). |
| `upstream-app4-disabled.conf` | Nginx | Upstream definition routing to only app1, app2, app3 nodes (app4 excluded). |
| `s3-proxy.conf` | Nginx | Defines `upstream s3proxy` with primary (`image-service.s3.amazonaws.com`) and backup (`image-service-west.s3.amazonaws.com`) S3 endpoints. |
| `supervisord.conf` | INI | Supervisord process manager config: runs 12 Python `imageservice.py` workers on ports 8000–8011, log rotation, PID file at `/var/groupon/log/image-service/`. |
| `uat/config.yml` | YAML | UAT environment client registry (same structure as production `config.yml` but with UAT-specific overrides). |
| `uat/nginx.conf` | Nginx | UAT environment nginx config: serves `img.grouponcdn.com`, `image-service-cache1-uat.snc1`, `image-service.local`; smaller proxy buffers than production. |
| `uat/upstream.conf` | Nginx | UAT upstream: two ports on `ims-demo1-uat.snc1` (ports 39170, 39171). |
| `uat/upstream-disabled.conf` | Nginx | UAT upstream with 12 ports on `image-service-app1-uat` (ports 8000–8011), used during UAT restarts. |
| `Capfile` | Ruby/Capistrano | Deployment task definitions for all environments (production, staging, UAT). |

## Secrets

| Secret Reference | Purpose | Store |
|-----------------|---------|-------|
| `s3.secret_access_key` (in `config.yml`) | AWS S3 secret access key for image bucket authentication | File-based (in `config.yml` — should be rotated to a secrets manager) |
| `s3.access_key_id` (in `config.yml`) | AWS S3 access key ID | File-based (in `config.yml`) |
| `clients.*.api_key` (in `config.yml`) | Per-client API keys for request authentication | File-based (in `config.yml`) |

> Note: AWS credentials and client API keys are stored directly in `config.yml` as plaintext. This is a known security concern — these should be migrated to a secrets management system.
