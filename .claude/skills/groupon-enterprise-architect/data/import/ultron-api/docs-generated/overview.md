# Ultron API Overview

## Purpose

ultron-api is a Scala/Play Framework service that provides a centralized job scheduling and management platform for Groupon's data operations. It manages the full lifecycle of job definitions, execution instances, resources, groups, permissions, and teams. The service integrates a Quartz-backed scheduler to trigger and monitor jobs, and an email-based watchdog system that alerts stakeholders when jobs fail to execute within expected time windows.

## Scope

### In scope

- Job definition CRUD and query operations (metadata, scheduling rules)
- Job instance lifecycle tracking (registration, state transitions, history)
- Resource, resource type, and resource location management
- Group, user, role, and permission management
- Team management
- Quartz-based job scheduling and watchdog monitoring
- Watchdog alert email dispatch via SMTP
- Resource lineage graph construction and data dictionary endpoints
- HTML UI views for operators to manage jobs via browser
- Evolution (database schema version) and health check endpoints

### Out of scope

- Executing the actual job workloads (handled by job runner clients calling into ultron-api)
- Storing job output data or analytics results (handled by downstream data pipelines)
- Email delivery infrastructure (delegated to the SMTP email service)
- Authentication and authorization enforcement beyond group/role/permission management

## Domain Context

- **Business domain**: Data Operations / Job Orchestration
- **Platform**: Continuum
- **Upstream consumers**: Job runner clients (`jobRunnerClients_4c7b`) that register job runs and watermarks
- **Downstream dependencies**: `continuumUltronDatabase` (primary metadata store), `continuumQuartzSchedulerDb` (scheduler state), SMTP Email Service (`smtpEmailService_2d1e`)

| Role | Description |
