# Robots.txt Request Serving Flow

## Summary

When a search engine crawler requests `/robots.txt` from any Groupon TLD, the `routing-service` forwards the request to `continuumSeoLocalProxyNginx`. Nginx determines the country and brand from the `X-Forwarded-Host` header, then proxies the correct robots.txt file from `continuumSeoLocalProxyS3Bucket` via the Hybrid Boundary endpoint. The robots.txt file controls which paths search engines are allowed to crawl on the corresponding Groupon storefront.

## Trigger

- **Type**: api-call
- **Source**: `routing-service` forwarding requests matching `/robots.txt` from `https://www.groupon.{tld}/`
- **Frequency**: per-request (on demand; search engines typically fetch robots.txt daily or on first visit)

## Participants

| Participant | Role in Flow | Architecture Ref |
|-------------|-------------|-----------------|
| routing-service | Routes `/robots.txt` requests to the Nginx proxy | `routingService` |
| SEO Local Proxy Nginx | Resolves country/brand; constructs S3 proxy path for robots.txt | `continuumSeoLocalProxyNginx` |
| Request Router | Nginx map directives that derive `$country`, `$website`, `$environment` | `requestRouter` |
| S3 Proxy | Nginx `proxy_pass` to Hybrid Boundary endpoint for robots path | `s3Proxy` |
| SEO Local Proxy S3 Bucket | Stores and serves the robots.txt file | `continuumSeoLocalProxyS3Bucket` |

## Steps

1. **Search engine requests robots.txt**: A crawler sends `GET https://www.groupon.fr/robots.txt`.
   - From: Search engine crawler
   - To: Groupon TLD edge / `routing-service`
   - Protocol: HTTPS

2. **routing-service forwards to Nginx**: `routing-service` routes the request to `seo-local-proxy--nginx`, setting `X-Forwarded-Host: www.groupon.fr`.
   - From: `routingService`
   - To: `continuumSeoLocalProxyNginx`
   - Protocol: HTTP

3. **Nginx resolves country and brand**: The `requestRouter` applies `map` directives:
   - `$country` resolves to `FR` (from `www.groupon.fr` TLD match)
   - `$website` resolves to `https` (default for groupon.* domains)
   - `$environment` resolves to `production`
   - From: `requestRouter`
   - To: internal Nginx variable resolution
   - Protocol: Nginx map

4. **Nginx matches the robots.txt location block**: Nginx applies:
   ```
   location /robots.txt {
     proxy_pass http://seo-local-proxy.$environment.service/robots/$country/$website/robots.txt;
     proxy_set_header Cookie "";
   }
   ```
   The resolved proxy URL becomes: `http://seo-local-proxy.production.service/robots/FR/https/robots.txt`
   - From: `s3Proxy`
   - To: `continuumSeoLocalProxyS3Bucket` (via Hybrid Boundary endpoint)
   - Protocol: HTTP (proxy_pass)

5. **S3 returns the robots.txt file**: The Hybrid Boundary endpoint serves `/robots/FR/https/robots.txt` from S3.
   - From: `continuumSeoLocalProxyS3Bucket`
   - To: `continuumSeoLocalProxyNginx`
   - Protocol: HTTP

6. **Nginx returns response to caller**: Nginx streams the robots.txt content back through `routing-service` to the crawler. Cookies are stripped.
   - From: `continuumSeoLocalProxyNginx`
   - To: Search engine crawler
   - Protocol: HTTPS

## Error Handling

| Error Scenario | Handling Strategy | Outcome |
|---------------|-------------------|---------|
| robots.txt missing in S3 for country/brand | S3 returns 404; Nginx forwards 404 | Crawler may block or unblock all paths depending on fallback behaviour |
| Hybrid Boundary unreachable | Nginx returns 502/504 | Crawler receives 5xx; retries on next session |
| `X-Forwarded-Host` header absent | Nginx defaults: `$country=US`, `$website=https` | US Groupon robots.txt served (wrong content for non-US TLD) |
| Wrong robots.txt content served | Configuration or header mapping error | Requires nginx config fix and redeploy (see Runbook) |

## Notes on robots.txt Content

- robots.txt files are generated by the cron job from templates: `robots.production.{country}.{brand}.txt` in `groupon-site-maps/config/`.
- Any change to robots.txt content requires updating the `groupon-site-maps` repository and redeploying the cron job image to regenerate and upload.
- Robots.txt changes must be validated via Google Search Console's robots.txt checker before deployment.

```
SearchEngineCrawler -> RoutingService: GET https://www.groupon.fr/robots.txt

S3Proxy -> S3Bucket: GET http://seo-local-proxy.production.service/robots/FR/https/robots.txt
S3Bucket --> S3Proxy: 200 OK (robots.txt content)
S3Proxy --> NginxProxy: 200 OK (strip cookies)
NginxProxy --> RoutingService: 200 OK
RoutingService --> SearchEngineCrawler: 200 OK (robots.txt)
```
