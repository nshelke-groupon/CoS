# API Surface

## Overview

CLO Inventory Service exposes two primary REST API groups via Jersey/JAX-RS resources within Dropwizard: an Inventory API for managing CLO products, merchants, reservations, units, and user rewards, and a Consent API for managing consent records, consent history, and card enrollments. All endpoints follow standard Groupon JTIER conventions and are documented via Swagger. API consumers are primarily internal CLO platform services and consumer-facing applications.

## Endpoints

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/clo/consent/{userId}` | Retrieve consent record for a user | JTIER auth |
