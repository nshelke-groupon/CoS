# Deployment

## Overview

MPP Service V2 is deployed as a Docker container orchestrated by Kubernetes across multiple GCP and AWS regions. Container images are built from `src/main/docker/Dockerfile` using the `prod-java17-jtier` base image and published to `docker-conveyor.groupondev.com/merchantpages/mpp-service-v2`. Deployment manifests are managed by the JTier Raptor deployment framework via `.meta/deployment/cloud/`. CI/CD runs on Jenkins via `Jenkinsfile` using the `java-pipeline-dsl` shared library. The service exposes HTTP on port 8080, admin on 8081, and JMX on 8009.

## Infrastructure

| Component | Technology | Details |
|-----------|-----------|---------|
| Container | Docker | `src/main/docker/Dockerfile` (base: `docker.groupondev.com/jtier/prod-java17-jtier:2023-12-19-609aedb`) |
| Orchestration | Kubernetes | Manifests generated by JTier Raptor from `.meta/deployment/cloud/components/app/` YAML files |
| Registry | Docker Conveyor | `docker-conveyor.groupondev.com/merchantpages/mpp-service-v2` |
| Load balancer | Kubernetes Service (internal) + Hybrid Boundary proxy | HTTP port 8080 exposed; admin port 8081 and JMX port 8009 via `exposedPorts` |
| CDN | Not applicable | Merchant page content is served directly via the Hybrid Boundary / internal VIPs |

## CI/CD Pipeline

- **Tool**: Jenkins
- **Config**: `Jenkinsfile` (uses `@Library("java-pipeline-dsl@latest-2")`)
- **Trigger**: Push to `main` branch or `MBNXT-30852` branch

### Pipeline Stages

1. **Build and test**: Runs `mvn clean verify` inside the CI Docker container (`docker compose -f .ci/docker-compose.yml run test`)
2. **Docker build**: Builds service Docker image
3. **Deploy to staging**: Deploys to `staging-us-central1` and `staging-europe-west1`
4. **Deploy to production**: Deploys to `production-eu-west-1` (as configured in `Jenkinsfile` `deployTarget`)
