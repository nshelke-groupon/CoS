# Deployment

## Overview

Goods Promotion Manager is containerized using Docker and deployed to Google Cloud Platform (GCP) using Kubernetes, orchestrated by Groupon's Conveyor/Raptor platform. Two active environments exist: `staging-us-central1` (GCP `stable` VPC) and `production-us-central1` (GCP `prod` VPC). Deployments are managed via Deploybot, which triggers automatically on merge to `master` and promotes artifacts from staging to production through a manual promotion step.

## CI/CD Pipeline

- **Tool**: Jenkins (cloud-jenkins) + Deploybot
- **Config**: `Jenkinsfile`, `.deploy_bot.yml`
- **Trigger**: Push/merge to `master` branch triggers Jenkins build and Deploybot staging deployment automatically

### Pipeline Stages

1. **Build and Test**: Maven builds the project (`mvn clean verify`), runs unit and integration tests, and produces a Docker image tagged `docker-conveyor.groupondev.com/inventory/goods-promotion-manager`
2. **Deploy to Staging**: Deploybot automatically deploys the new image to `staging-us-central1` (`goods-promotion-manager-staging-sox` namespace) via `bash ./.meta/deployment/cloud/scripts/deploy.sh staging-us-central1 staging goods-promotion-manager-staging-sox`
3. **Smoke Test**: Engineer verifies functionality in staging; Kibana logs available at `stable-kibana.us-central1.logging.stable.gcp.groupondev.com`
4. **Promote to Production**: Engineer clicks "Promote" in Deploybot; Deploybot creates a GPROD ticket and runs `bash ./.meta/deployment/cloud/scripts/deploy.sh production-us-central1 production goods-promotion-manager-production-sox`
5. **Production Verification**: Engineer checks Wavefront dashboards and logging for error-free traffic at normal processing times

**Ports:**
- `8080` — HTTP application traffic (exposed as port 80 on the Kubernetes service)
- `8081` — JTier admin port
- `8009` — JMX port

**Known deployment gotcha:** The Quartz job scheduler acquires many database connections on startup, which can occasionally cause smoke tests to fail. If rollback occurs due to a smoke test failure, wait at least one hour before retrying the deployment. This is a known Quartz/PostgreSQL interaction and does not indicate a code defect.
