# Coupons Commission DAGs Overview

## Purpose

coupons-commission-dags provides Apache Airflow DAGs that orchestrate coupons commission reporting data pipelines running entirely in Google Cloud Platform. Each DAG provisions an ephemeral GCP Dataproc cluster, submits a Spark job that performs a specific pipeline stage (sourcing, transformation, or aggregation), and then tears the cluster down. The service covers two reporting cadences: a monthly pipeline for all affiliate networks and a daily pipeline for the Awin affiliate network.

## Scope

### In scope

- Airflow DAG definitions for all six commission reporting pipeline stages
- Cluster lifecycle management: creating and deleting ephemeral Dataproc clusters per run
- Spark job submission to Dataproc via `DataprocSubmitJobOperator`
- Per-environment JSON configuration for DAG parameters, cluster specs, and Spark JAR URIs
- Scheduling of monthly (1st of month) and daily (every day) pipeline runs
- Multi-account and multi-network commission reporting for affiliate networks: Impact Radius (IR), Awin, Partnerize, eBay, Skimlinks, CommissionFactory, BrandReward, CJ, Webgains, TradeDoubler, Pepperjam

### Out of scope

- The Spark job business logic (sourcing, transformation, aggregation) — implemented in separate JARs deployed to Groupon Artifactory
- Storage of commission data — managed by the Spark jobs themselves using the Dataproc Metastore
- Commission calculation rules or affiliate network API integration — handled within the Spark JARs
- Airflow cluster management — provided by GCP Cloud Composer

## Domain Context

- **Business domain**: Coupons / Finance / Affiliate Commission Reporting
- **Platform**: Continuum
- **Upstream consumers**: Finance and accounting teams consuming commission reports; no upstream service triggers these DAGs programmatically (schedule-driven)
- **Downstream dependencies**: GCP Dataproc (cluster execution), GCP Dataproc Metastore (Hive metadata), Groupon Artifactory (Spark JAR artifacts)
