# CKOD Worker Overview

## Purpose

CKOD Worker is a scheduled Python worker service that orchestrates SLA lifecycle management, data pipeline monitoring, incident management, and AI-assisted deployment operations for the Continuum platform. It runs a set of cron and interval jobs via APScheduler to continuously evaluate pipeline health, track SLA compliance, manage deployment approvals, and automate operational responses. When anomalies are detected it raises incidents in Jira/JSM and sends alerts to Google Chat.

## Scope

### In scope

- SLA entry initialization and lifecycle updates for Keboola, Resource Manager, Optimus Prime, and MDS pipeline flows
- Monitoring and alerting for failing, warning, and long-running pipeline runs
- Deployment operations: approval checks, completion checks, and deployment window notifications
- Synchronizing pipeline run metadata and cost data from Keboola
- Agent polling: querying CKOD UI for pending tasks and invoking Vertex AI reasoning-engine sessions for automation
- Synchronizing user/group authorization data from Service Portal
- Sending operational notifications to Google Chat webhooks
- Creating, updating, and transitioning Jira Service Management tickets for incidents and deployments

### Out of scope

- Running or executing data pipelines (owned by Keboola and Optimus Prime)
- Incident response resolution (owned by on-call engineers; CKOD Worker opens and tracks tickets only)
- Serving user-facing HTTP requests (worker only; UI is owned by `continuumCkodUi`)
- Storing raw pipeline or job execution data (sourced from `servicePortal`, `optimusPrime`, `keboola`)
